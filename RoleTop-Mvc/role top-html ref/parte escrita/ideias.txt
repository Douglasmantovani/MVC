oque o cliente precisa

-um sistema de login(campo com email e senha e um captcha )
-sistema de cadastro(campos com informaçoes/dados pessoais endereço de email e dados do cartao de credito como opcional)
-sitema de avaliaçao para haver transparencia com cliente(o algoritimo somara as avaliaçoes e calculara a media) 
-sistema de pagamento seguro,que proteja os dados do cliente (sistema de pague seguro https://enotas.com.br/blog/pagseguro-como-funciona/)
-sistema de termos de liceça(contrato on line para que o cliente esteja ciente das especificaçoes e deveresque tem com o local) 
-sitema de devoluçao para extornar o dinheiro do cliente em caso de cancelamentos(havera uma avaliaçao de uma pessoa fisica para evitar fraude,e avaliara se o caso do extorno é legitimo) 
-sistema para reclamaçoes(um email para reclamçoes)
-aba de que detalhe o local(banheiros,saidas de emergencia) e sua localizaçao(estara no contrato) 
-filtros para que seja mais facil achar eventos dentro das especificaçoes do cliente(filtro de multiplas escolhas que possiblilete combinaçoes)
-sistema de agendamento que indique datas e horarios disponiveis
-aba de comentarios (area para que os cadastrados tirem suas duvidas e compartilhem informaçoes e experiencias)
-sistema contra bots maliciosos (eu nao sou robo/captcha)
-aba de contato para haver comunicaçao com os administradores do local(email e numero de telefone especifico para entar em conato)
-linguagem formal e objetiva para que nao haja duvidas
-sistema de banco de dados(SQL Server ums dos mais recomendados para e-comerce)
